HTTP：基本概念

作者: 呼佳慧
发表于2025-10-21
更新于2025-10-21

HTTP，即超文本运输协议，是实现网络通信的一种规范。

更具体地说，HTTP是专门进行两点之间传输文字图片音频等超文本数据的约定和规范

具有[支持客户/服务器模式，简单快速，数据类型灵活，无连接，无状态]的特性。

HTTP与HTTPS

HTTP信息是明文传输，存在安全风险的问题。
HTTPS加入了 SSL/TLS 安全协议，使得报文能够加密传输。

HTTPS 在 TCP 三次握手之后，还需进行 SSL/TLS 的握手过程

HTTP 默认端口号是 80，HTTPS 默认端口号是 443。

HTTPS 协议需要向 CA申请数字证书，来保证服务器的身份是可信的

小结
  1. 加密传输
  2. SSL/TSL握手
  3. 端口号修改
  4. 数字证书

HTTP状态码

  1. 状态码 含义 常见值和含义
    1XX 提示信息, 协议处理的中间状态,很少使用
    2xx 服务器成功处理了客户端请求 [ 200 ok ] 一切正常
    [ 204 no content] 没有body数据
    [ 206 partial content] 分块下载或断点续传
    3xx 客户端请求的资源发生了变动,需要客户端用新的url重新发送请求获取, 即 重定向 [ 301 moved permanently] 永久重定向,资源已经不存在了
    [ 302 found ] 临时重定向,资源存在,但是需要另一个url访问
    「304 Not Modified」不具有跳转的含义，表示资源未修改，重定向已存在的缓冲文件
    4xx 表示客户端发送到报文有误,服务器无法处理,错误码 [ 400 bad request ] 报文有错
    [ 403 Forbidden ] 服务器禁止访问资源
    [ 404 not found ] 请求的资源在服务器上不存在未找到
    5xx 报文正确,但是 服务器处理事内部发生了错误,是服务端的错误码 500 通用
    [ 501 not implemente] 功能还不支持
    [ 502 bad gateway ] 服务器作为网关或者代理时返回的错误码,自身正常,后端服务器发生错误
    [ 503 service unavailable ] 服务器很忙暂时无法响应

HTTP缓存

强制缓存和协商缓存。

强制缓存

浏览器判断缓存没有过期,就直接使用浏览器的本地缓存，过期了就请求服务器并返回新的。

  • cache-control : 相对时间 优先级更高
  • expires : 绝对时间

流程：

  1. 第一次访问资源,返回资源,cache-control：保质期
  2. 再次访问时, 浏览器计算是否过期,过期重新请求
  3. 再次收到后就更新
协商缓存

服务端告知客户端是否可以使用缓存，返回304。

即比强制缓存多一步，过期后问问服务端，改了么，没改就接着用。

法1

  • last-modified：响应头，本地资源离开服务端前最后一次修改的时间
  • if-modified-since：请求头，客户端再次请求时带上last-modified的值

服务端将资源上次修改的时间于if-modified-since对比，未修改则304.

法2

  • Etag：资源离开服务器前带的唯一标识符。
  • if-none-match：请求头，再次请求时带上Etag值。

服务端比较Etag是否变化，未变化则304。

优先级

基于唯一标识符Etag更加准确，Etag优先级更高。

HTTP/1.0

服务器完成请求处理后立即断开TCP连接，每次请求都需要与服务器建立一个TCP连接。这种形式明显造成了性能上的缺陷。

如果需要建立长连接，需要设置一个非标准的Connection字段 Connection: keep-alive

HTTP/1.1

默认支持长连接（Connection: keep-alive），即在一个TCP连接上可以传送多个HTTP请求和响应，减少了建立和关闭连接的消耗和延迟。

HTTP 1.1还允许客户端不用等待上一次请求结果返回，就可以发出下一次请求，但服务器端必须按照接收到客户端请求的先后顺序依次回送响应结果。

增加更多的请求头和响应头来完善的功能，如下：

  • 引入了更多的缓存控制策略，如If-Unmodified-Since, If-Match, If-None-Match等缓存头来控制缓存策略
  • 引入range，允许值请求资源某个部分
  • 引入host，实现了在一台WEB服务器上可以在同一个IP地址和端口号上使用不同的主机名来创建多个虚拟WEB站点

HTTP/2.0

而HTTP2.0在相比之前版本，性能上有很大的提升。

  • 多路复用：
    HTTP/2 复用TCP连接，在一个连接里，客户端和浏览器都可以同时发送多个请求或回应，而且不用按照顺序一一对应，这样就避免了”队头堵塞”
  • 二进制分帧：
    采用二进制格式传输数据，而非 HTTP 1.x的文本格式，解析起来更高效。
  • 首部压缩：
    使用“首部表”来跟踪和存储之前发送的键值对，对于相同的数据，不再通过每次请求和响应发送
  • 服务器推送
