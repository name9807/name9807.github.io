VUE：响应式原理

写于24年9月，25年10月重写。

响应性是一种可以使我们声明式的处理变化的编程范式。

即，更新并非由程序员使用代码来主动刷新，而是由后面隐藏机制来负责维护刷新。

最简单的响应性例子：

若我们通过某种声明式的语句，建立A2与A0，A1的自动映射，就称具有响应性

”探究响应性原理 = 探究上面例子实现的思路“

原生 JavaScript 没有提供任何机制能来追踪局部变量的读写。但是，我们是可以追踪对象属性的读写的。

在 JavaScript 中有两种劫持 property 访问的方式：getter / setters 和 Proxies。

Vue 2 使用 getter / setters 完全是出于支持旧版本浏览器的限制。

而在 Vue 3 中则使用了 Proxy 来创建响应式对象，仅将 getter / setter 用于 ref。

Vue2

Vue 将遍历data对象所有的 property，并使用 Object.defineProperty 把这些 property 全部转为 getter/setter。

由于这个方式的限制，Vue2的响应式具有缺陷。

  • 初始化实例时才对 property 执行 getter/setter 转化：
    只有一开始就存在 data 中的 属性才是响应式的。
    弥补：Vue.set(object, propertyName, value)
  • 不能利用索引直接设置元素：
    数组索引具有特殊性，getter/setter是针对对象的具体属性定义的
    原生数组方法不触发 setter
    弥补：set方法或使用数组修改方法（重写了数组的 7 个原生方法

这些 getter/setter 对用户来说是不可见的，但是在内部它们让 Vue 能够追踪依赖，在 property 被访问和修改时通知变更。这里需要注意的是不同浏览器在控制台打印数据对象时对 getter/setter 的格式化并不同，所以建议安装 vue-devtools 来获取对检查数据更加友好的用户界面。

每个组件实例都对应一个 watcher 实例，它会在组件渲染的过程中把“接触”过的数据 property 记录为订阅者。之后当依赖项的 setter 触发时（作为事件，会通知 watcher，从而使它关联的组件重新渲染。

另外Vue2中的ref是用来绑定元素的。

在v2中，ref 被用来给元素或子组件注册引用信息。引用信息将会注册在父组件的 $refs 对象上。如果在普通的 DOM 元素上使用，引用指向的就是 DOM 元素；如果用在子组件上，引用就指向组件实例

Vue3

Vue 3 中则使用了 Proxy 来创建响应式对象，仅将 getter / setter 用于 ref。

ref的实现

原理和Vue2相同。在模板中使用 ref 时，我们不需要附加 .value。

reactive的实现

具有一定局限性：

  • 只能用于对象类型 (对象、数组和如 Map、Set 这样的集合类型)。它不能持有如 string、number 或 boolean 这样的原始类型。
  • 当你将一个响应式对象的属性赋值或解构到一个本地变量时，访问或赋值该变量是非响应式的，因为它将不再触发源对象上的 get / set 代理。注意这种“断开”只影响变量绑定——如果变量指向一个对象之类的非原始值，那么对该对象的修改仍然是响应式的。
